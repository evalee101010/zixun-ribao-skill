# 资讯类简报 · 排版润色规范

> 公司主营：出海泛娱乐。本简报聚焦三类内容——泛娱乐（出海优先）、娱乐相关创新（AI模型/应用、创新硬件）、全球融资。
> 当前排除方向：具身/机器人（人形、机械臂、四足、宇树等）现阶段先不看；"聊天机器人/对话机器人"等 AI 产品不在此列、照常保留。
> 用法：脚本 fetch_news.py 抓取出草稿（候选清单），按本规范把 ⭐精选 浓缩+排版，套用《日报模板.md》产出 .md 文件。
> 来源统一处理：**所有源一视同仁**走同一条流水线——抓取 → 关键词筛选 → 时间窗 → 去重 → 模板浓缩 → 配图规则。微信公众号经 WeWe RSS 接入（见 wewe-rss/），和白鲸/36氪/IT之家/ai-bot 等源**遵循完全相同的筛选与格式标准，不特殊对待**。应用侧数据走特定 API。

## 一、板块结构（三大一级板块）

| 一级板块 | 二级赛道 | 覆盖范围 |
|---|---|---|
| **泛娱乐·出海** | 休闲游戏 ／ 短剧/漫剧 ／ 小说·网文 ／ 社交·Dating ／ 其他 | 业务、资本、财报、数据披露，出海方向优先 |
| **创新·AI娱乐** | AI模型与应用（娱乐向）／ 创新硬件（娱乐向） | 与娱乐相关的创新形态 |
| **全球融资** | 泛娱乐融资 ／ 泛AI·AI应用融资 | 全球范围的融资/资本动向，集中收口 |

规则：
- 当日没有内容的赛道直接省略，不留空标题。
- **融资类一律归到「全球融资」板块**集中呈现，不分散到各赛道。
- 层级固定四级：`#`一级板块 → `##`二级赛道 → `### n）标题` → 正文；不跳级。序号用全角 `）`，每个赛道下从 `1）` 重新计数。
- 标题间距：不同 `#` 一级板块之间、不同 `##` 二级赛道之间，都用独立一行 `<br>` 做“真实回车式空行”；也就是上一板块/赛道最后一条链接结束后，空一行写 `<br>`，再空一行写下一个标题。
- 标题图标固定：一级板块使用 `# 🌍 泛娱乐·出海`、`# ✨ 创新·AI娱乐`、`# 💸 全球融资`；二级赛道使用 `## 🕹️ 休闲游戏`、`## 🎬 短剧/漫剧`、`## 📚 小说·网文`、`## 💬 社交·Dating`、`## 🌐 其他`、`## 🧠 AI模型与应用（娱乐向）`、`## 🎧 创新硬件（娱乐向）`、`## 🎭 泛娱乐融资`、`## 🧩 泛AI·AI应用融资`。

## 二、单条资讯格式

公共要求(两种子模板都适用):

- **标题**:一句话、简单但信息明确,不加类型标签。含公司/产品 + 核心事件,能带数字尽量带(如"Kuku TV 母公司冲刺 IPO,目标估值 15.7 亿美元")。
- **篇幅**:按原文信息量**忠实浓缩**,通常 **200–400 字**;信息多就写透、信息少就短,**不注水、不自行延展观点**。
- **只浓缩、不做阅读理解**:日报的本分是把原文已有的信息提炼出来(是什么、关键数据、各方说法),**不加入编辑自己的分析、判断或"解读/意义"**。原文里有的观点可以转述并注明出处(如"据XX分析"),自己不要凭空生成结论。
- **正文不写采集来源/兜底说明/内部过程话术**:单条资讯正文里不写“该条来自/经 WeWe RSS/VVRS/VVRSS/VBRS 抓取/来源为 XXX”这类采集渠道或 Source 说明；也不写“补足观察信号”“以原文为准”“待补全文”“草稿摘要显示”“公开摘要未展开”“该条作为……保留”“后续可继续跟踪”等解释采集不足、筛选原因或后续动作的内部过程话术。来源统一放文末 `Source 来源` 表。正文只保留新闻事实本身。
- **原文链接**:裸链接单独成行,放每条最后。**每条必带链接；URL 不做任何 Markdown 转义**，保持 `https://example.com/a-b.html` 这种可直接复制到 Chrome 打开的形式。
- **取全文**:优先基于完整正文浓缩,脚本已自动取全文——ai-bot/IT之家/36氪/TechCrunch 直接给正文;白鲸(静态`<p>`)/36氪出海(`__NEXT_DATA__`)由脚本解析;**公众号经 WeWe RSS 输出全文(已清理符号残渣,杜绝 `&&&&` 乱码)**。仅未接入 WeWe 的公众号才需人工贴链接/正文。若 WeWe RSS/公众号条目只拿到标题和链接,但仍属于 ⭐精选且应保留,只能写标题级事实,不得自行扩写观点或细节。
- **配图(仅两类情况)**:① 榜单/数据可视化(手游收入榜、短剧周榜等)配榜单图;② 以产品形态为亮点的资讯配产品截图。**其余条目不配图**,避免日报变图库。规则:每条最多 1 张,**从草稿给的"候选图"URL 里挑最具代表性的一张**(榜单挑那张榜单图、产品挑产品/界面截图),放正文之后、链接之前,写 `![](图片URL)`——直接用源站图 URL 即可,粘进飞书会自动拉取并转成飞书托管图。微信来源(ai-bot 那批)的图有防盗链,**不要用**。脚本没给到合适候选图时,就不配图。

根据信息维度多少,二选一:

### 模板 A:分维度标签型(业务/财报类首选,信息维度多时用)

适合产品/公司类业务资讯——用标签化小标题把信息拆成几个维度,每个维度一条 `-`。

```
### n）简明标题

- 产品概况：{是什么、玩法/形态、研发与发行方}
- 市场数据：{下载量／流水／榜单／RPD 等}
- {优化/动作}：{本期关键动作}
- 用户群体：{目标人群}

{原文链接}
```

常用标签(按内容取用,不必全上):产品概况 / 市场数据 / 财报数据 / 运营动作 / 产品优化 / 竞争格局 / 用户群体 / 行业影响。

### 模板 B:概述段落型(简单资讯、单点消息用)

```
### n）简明标题

{1–2 句概述：是什么、关键变化、为什么重要}

- 关键数据：{可量化信息}

{原文链接}
```

### 模板 C:融资型(「全球融资」板块专用)

两段纯叙述,不使用数据点要点:

```
### n）{公司}{轮次}融资标题，{核心亮点}

{日期}，{公司}宣布完成{金额}{轮次}融资。本轮股东包括{投资方}，{老股东/加码情况}。

{公司成立时间、定位/在做什么}。近期，{技术/业务/数据亮点，说明为何值得关注}。

{原文链接}
```

要点:
- 标题=公司 + 轮次/金额 + 一句亮点(如"千寻智能再获15亿元A+轮融资，自研具身模型登顶北美具身智能'奥林匹克'")。
- 第一段必须交代:日期、金额、轮次、投资方构成、老股东动向。
- 第二段交代公司背景 + 近期最值得说的进展(榜单/产品/数据),给出为什么重要。
- 金额/估值标注币种,出海项目给本币与美元折算。

## 二之二、时间维度 · 精选排序 · 去重（每日必做）

**1. 时间维度**
- 覆盖范围固定为**北京时间 前一日16:00 — 当日16:00**。
- 标题下方注明：`> 时间范围：{前一日}16:00 — {当日}16:00（北京时间）`。
- 抓取脚本已按此窗口过滤带时间戳的源；列表页（白鲸/36氪出海）无时间戳、标 ⏱，并按“无时间戳链接仅首次出现保留”做历史去重，仍需人工核对是否落在窗口内，超窗的剔除。
- 抓取脚本会自动回溯前 2 天草稿/成稿链接；若 URL 已在前 2 天出现，本日不再新增。

**2. 精选与排序（控篇幅）**
- 不求全，求精。每个二级赛道取**最重要的几条**（默认 Top7），整篇**建议 ≤ 30 条**，避免冗长。
- 排序/取舍优先级：① 出海/全球方向；② 有具体数据或资本动作（融资额、榜单、流水、增长、估值/IPO）；③ 头部公司或重点主体（腾讯/字节/快手/米哈游/9yards 等）；④ 时效更新。
- 脚本已按上述信号打分（score）并排出 ⭐精选 与「备选」；篇幅紧张时按 score 由高到低保留，备选仅作补充池。
- 成稿前必须做 ⭐精选覆盖校验：草稿中每条 ⭐精选 要么进入成稿，要么因跨源/跨日重复、超出时间窗、明显无关或信息质量不足而剔除；WeWe RSS/公众号条目即使只有标题+链接，也不能无声跳过，只能保留标题级事实并避免扩写。

**3. 去重**
- 同一条新闻多源命中时只保留信息最全的一条（优先带数据/带摘要者），其余合并或舍弃。
- 脚本已做四类去重：本次抓取内链接精确去重、跨源标题近重复（2-gram 含入度≥0.7）、前 2 天 URL 回溯去重、无时间戳链接首次出现去重；成稿时再人工核一遍主题级重复（同一事件不同标题）。

## 三、润色规则

- 信息密度优先，去口水；能给数字就给数字。
- 客观中性，分析性表述注明出处（"据XX报道""分析指出"）。
- 时间口径固定为「昨日16:00—当日16:00」，措辞统一为"6月X日""近日/日前"。
- 出海/全球内容标注地区与币种（如"约15.7亿美元""250亿至350亿卢比"）。

## 四、字符与转义约定

- 标题/正文中的英文句点、连字符可按需转义：`.` → `\.`，`-` → `\-`（与原件一致，飞书/Markdown 渲染更稳）。
- **URL 绝不转义**：原文链接、飞书 wiki 链接、图片 URL 都保持裸链；不要写成 `https://www\.qbitai\.com/...`，否则复制到 Chrome 会变成非法 hostname。
- 中文标点用全角；百分比、区间照常写（`10%-20%`、`2.6亿至3.6亿美元`）。

## 五、Source 来源（固定收尾）

正文结束后保留分隔线 `---`，然后放本期实际采用的来源统计，不再放“个人社媒”“兴趣产品跟踪”与 appendix。

```
---

Source 来源：

|来源|本期采用条数|
|---|---:|
|白鲸出海|3|
|IT之家|2|
|DataEye短剧出海公众号（经 WeWe RSS）|1|
```

规则：
- 只统计最终成稿正文中实际采用的来源，不统计备选、抓取失败源、过滤掉的重复源。
- 来源名称尽量沿用草稿中的 `｜来源`，公众号可写为“公众号名（经 WeWe RSS）”或“公众号名（经 AI工具集每日AI资讯）”。
- 条数按成稿正文条目计数；同一条引用多个来源时，主来源计 1 条。

## 六、产出与命名

- 输出 Markdown 文件，命名：`资讯类简报-{月.日}日.md`。
- 当前手动触发；如需固定时间自动生成，再单独配置。
